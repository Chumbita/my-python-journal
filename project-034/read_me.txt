Proyecto 34
En este pequeño proyecto seguí trabajando con sockets. La diferencia que tiene este programa con respecto a los dos anteriores es que el servidor es capaz de antender a varios clientes. En los ejemplos anteriores cuando un cliente se conectaba con el servidor, este último se bloqueaba y si otros clientes se querian conectar con el servidor estos entraban en una cola de espera. Ahora haciendo uso de la librearía threading creamos un hilo por cada cliente que se conecta y de esta manera el servidor es capaz de antender a múltiples clientes de manera simultánea. Los clientes al conectarse pueden enviar mensajes hasta que estos ingresen la palabra reservada que los desconecta. Por otro lado el servidor imprime la información de los clientes como lo es la dirección y el mensaje que se recibió por parte de este. 


Project 34
In this small project, I continued working with sockets. The main difference between this program and the previous two is that the server is now able to handle multiple clients simultaneously. In the earlier examples, when a client connected to the server, the server would block, and any other clients trying to connect would be placed in a queue. Now, by using the **threading** library, we create a thread for each client that connects, allowing the server to serve multiple clients concurrently. Clients can send messages until they enter a reserved word that disconnects them. Meanwhile, the server prints information about the clients, such as their address and the message it received from them.
